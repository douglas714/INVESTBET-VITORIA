============================================================
 INVESTBET - INSTALACAO RAPIDA
============================================================

PASSO 1: PREPARACAO
====================

1. Certifique-se de que tem:
   - Node.js 18+ instalado (https://nodejs.org/)
   - MySQL/XAMPP rodando
   - Arquivo .env.template presente

2. Se nao tem .env.template, crie um arquivo chamado .env com:

   # The Odds API Configuration
   THE_ODDS_API_KEY=3a50d923ce7f50b649c36ced65029efd

   # Database Configuration
   DATABASE_URL=mysql://root:@localhost:3306/sports_betting

   # OAuth Configuration
   OAUTH_SERVER_URL=
   JWT_SECRET=local-dev-secret-key-change-in-production

   # Analytics
   VITE_ANALYTICS_ENDPOINT=
   VITE_ANALYTICS_WEBSITE_ID=

   # App Configuration
   VITE_APP_ID=sports-betting-robot
   VITE_APP_TITLE=Sports Betting Robot
   VITE_APP_LOGO=

   # OAuth Portal
   VITE_OAUTH_PORTAL_URL=

   # Owner Info
   OWNER_OPEN_ID=
   OWNER_NAME=

   # Forge API
   BUILT_IN_FORGE_API_URL=
   BUILT_IN_FORGE_API_KEY=
   VITE_FRONTEND_FORGE_API_KEY=
   VITE_FRONTEND_FORGE_API_URL=


PASSO 2: INSTALACAO
====================

1. Duplo clique em: install.bat

2. Aguarde o script completar (2-5 minutos)

3. O navegador abrira automaticamente em: http://localhost:3000

4. Se o navegador nao abrir, abra manualmente:
   - Digite no navegador: http://localhost:3000
   - Pressione Enter


PASSO 3: USAR O ROBO
====================

1. Clique em "INICIAR ANALISE"

2. Aguarde 30-60 segundos

3. Sistema mostrara os 6 melhores jogos para apostar

4. Clique em "Atualizar Resultados" para ver status dos jogos


PROBLEMAS COMUNS
================

Problema: "ERRO: Arquivo .env nao foi criado"
Solucao: Crie manualmente um arquivo .env com o conteudo acima

Problema: "0 Jogos encontrados"
Solucao: 
- Verifique se THE_ODDS_API_KEY esta no .env
- Verifique se a API nao atingiu limite (500 requisicoes/mes)

Problema: "MySQL nao esta rodando"
Solucao:
- Abra XAMPP Control Panel
- Clique em "Start" ao lado de "MySQL"
- Execute install.bat novamente

Problema: "Porta 3000 ja esta em uso"
Solucao:
- Feche outros programas que usam porta 3000
- Ou adicione PORT=3001 no arquivo .env


PROXIMAS VEZES
==============

Para iniciar o servidor sem reinstalar:

1. Duplo clique em: start.bat

2. Aguarde 15-30 segundos

3. Acesse: http://localhost:3000
